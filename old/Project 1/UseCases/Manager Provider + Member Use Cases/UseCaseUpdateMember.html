Use Case: Update Member extends Manage Member Records

Actors: Operator

Context: The Update Member use case allows an operator to update member information in the ChocAn data system.

Main Success Scenario:

  1. The system prompts the operator to enter a Member number.

      1.1 The operator enters a Member number.

  1. The terminal displays that the Member number is “valid.”
  2. The terminal displays the member’s current information including the following data fields:
  1. Member name (25 characters)
  2. Member street address (25 characters)
  3. Member city (14 characters)
  4. Member state (2 letter abbreviation)
  5. Member ZIP code (5 digits)
  1. The operator changes some of the data fields.
  2. The system prompts the operator to confirm their changes.

              5.1. The operator confirms the changes.

  1. The system records the updates.
  2. The terminal displays “Member Updated”.

Extensions: 

      2a. The operator entered an invlaid member number.

             2a.1. The terminal informs the operator and prompts the operator to try again

  2a.1.1. The operator selects to attempt again.

             2a.2. The use case continues at step 1.

                  2a.1.1a. The operator selects not to attempt again and the use case terminates

                   with an error.

      5.1a. The operator selects to cancel the update.

             5.1a.1 The use case is terminated with an error.

      6a. The operator enters invalid information

     6a.1. The system informs the operator

             6a.2. The use case continues at step 3.
