Use Case: Manage Member Records

Actors: Operator

Context: The Manage Member Records use case adds new Members, deletes Members who have resigned, and updates Member records.

Main Success Scenario: 

  1. The operator accesses the terminal.
  2. The terminal prompts the operator to “Add Member,” “Delete Member,” or “Update Member Information”.
  3. The operator selects “Add Member.”

      3.1 The system initiates the Add Member use case.

  1. The terminal displays “Member Record Change Successful”.

Extensions: 

      1a. The Software Product is not in interactive mode

              1a.1.     The Software Product informs the operator that the Software Product is not

       in Interactive mode.

              1a.2.     The use case terminates with an error.

      3a. The operator selects “Delete Member.”

     3a.1.  The system initiates the Delete Member use case.

3b. The operator selects “Update Member.”

           3b.1.  The system initiates the Update Member use case.

4a. The member record update fails.

             4a.1.  The terminal displays “Member Record Update Unsuccessful”.

             4a.2.  The use case terminates with an error.
