Use case: Manage Reports

Context: The Manage Reports use case allows managers to produce reports at any time during the week.

Actors: Manager

Main Success Scenario:

  1. The manager accesses the manager’s terminal.
  2. The terminal displays a menu of options.
  3. The manager selects “Produce Member Report”.
  4. The system prompts the manager to enter the member number.

               4.1. The manager enters the member number.

  1. The Produce Member Report use case is initiated.
  2. The report is displayed on the terminal.
  3. The terminal displays the option to print.

    7.1. The manager selects to print the report.

  1. The system prints the physical report.
  2. The terminal returns to the menu of options.
  3. The manager exits the terminal.

Extensions:

     3a. The manager selects “Produce Provider Report”

        3a.1. The system prompts the manager to enter the provider number.

        3a.2. The manager enters the provider number.

        3a.2. The Produce Provider Report use case is initiated.

     3b. The manager selects “Produce Summary Report”

        3b.1. The Produce Summary Report use case is initiated.

     6a. An error occurs during report production

        6a.1. The use case is terminated with an error.

     7.1a. The manager selects not to print the report.

             7.1a.1 Skip to Step 8.

     10a. The manager makes no selection.

             10a.1 The terminal exits after an amount of time of inactivity.
