Use Case: Verify Member Status

Context: The “Verify Member Status” use case allows a provider to access the member status of a member via the member’s unique ID.

Actors: Provider

Main Success Scenario: 

  1. The member’s card is swiped through the terminal by the provider, and the member’s unique ID number is read by the software product.
  2. “Member Valid” appears on the terminal.

Extensions: 

   

2a. The member’s status is suspended.

        2a.1. “Member Suspended” is displayed to the terminal.

        2a.2. The use case is terminated with an error.

2b. The member number is invalid.

        2b.1. “Invalid number” is displayed to the terminal.

        2b.2. The use case is terminated with an error.
